### 微信发消息，第二大脑就收到，他是这么一个流程

其实，核心就是对接了一个 **WeChat Bot（iLink）**。

```
你的手机微信
    │
    │  发一条消息：链接 / 想法 / 问题 / 截图
    ▼
微信官方 iLink Bot 平台（ilinkai.weixin.qq.com）
    │
    │  HTTPS 长轮询
    ▼
Obsidian CC 插件（跑在你电脑里）
    │
    │  共享：引擎 / 技能 / 工作区 / 写入确认门
    ▼
Agent 处理 → 写库 / 查库 / 沉淀
    │
    ▼
微信回你处理的中间结果和最后结果
```

![[01_raw/_inbox/文章/images/c3df2ca9f7a6f3548bfed4a75ceae44e_MD5.png]]

通过微信 Bot 和 Obsidian cc 连接起来

操作非常简单，安装 Obsidian-cc 之后，手机微信扫一次码授权，**就接通了**。

举几个真实我用到的场景：

```
地铁上看到一篇好文章
   你给微信 Bot 转发链接 → Agent 拉取正文 → 消化成候选记忆
   → 微信回你："✅ 已沉淀到 memory/inbox/，核心 3 条：……"

☕ 排队买咖啡，脑子里冒出一个想法
   你给微信 Bot 发一句话 → Agent 判断是灵感还是闲聊
   → 灵感就写进 raw/ideas/，闲聊就正常回
```

**所有这些，你只需要做一件事——掏出手机，给微信发一条消息。**

不用拿到电脑在操作。不用解锁，不用切 App，不用记任何前缀指令，在微信中就可以随时用上你的 Obsidian-cc

### 为什么这次不是"微信转发到自己邮箱"那种玩具

这次的关键不是"消息能传过去"，**关键是接进了一个真正能干活的 Agent**。

```
Obsidian CC 的微信 Bot：
   微信 → iLink → Agent（共享 13 个技能 / 工作区 / 确认门）
   ↑
   Agent 自己判断意图、调技能、写对应目录、回你总结
```

这里有几个**容易被忽略但极其关键的设计**：

**一，Agent 自己判断意图，不靠前缀匹配。**

很多 Bot 是这么做的：你说 `/write xxx` 就写，你说 `xxx` 就不写。这很蠢——因为人跟人聊天不会带前缀。

Obsidian CC 不是。微信 Bot 的 system prompt 里塞了一段微信端 persona：

```
你正通过微信公众号 Bot 与用户在手机微信里对话。
- 回复要像微信聊天一样简短、口语化、能直接扫读。
- 长内容（整理后的笔记、记忆沉淀、周回顾）写进 Vault
  工作区文件，对话里只回一句话总结 + 文件位置，
  绝不把整篇内容贴进对话。
- 意图模糊时问 1～2 个关键问题；意图明确要沉淀时
  直接归位；纯闲聊就正常回应。
```

Obsidian 会记住你之前的偏好，你最近在做什么，Agent 自己判断这是要记下来还是要继续聊。**不需要你背指令**。

**二，微信会话和桌面端会话完全隔离。**

```
桌面端对话面板：session key = 
微信 Bot：      session key = wx:wechat
```

![[01_raw/_inbox/文章/images/20cf39f1c947125749762dbced9821c7_MD5.png]]

独立会话

两个 session 各自独立——上下文不串、transcript 不串、快照不串。你在地铁上跟微信 Bot 聊了一通，回家打开桌面端对话面板，那边的对话不会被污染。

**但它们共享同一套底层**——同一个引擎、同一套 13 个技能、同一个工作区、**同一扇写入确认门**。

### 几个细节，体现这次做得有多扎实

讲完主架构，再讲几个我觉得值得展开的细节——

**二维码本地生成。** 设置页「微信」里点「扫码 / 管理连接」，弹出来的二维码是**插件本地用 qrcode 这个 npm 包生成的**，不联网取图。意思是——你的扫码流程不经过任何第三方图片服务。

**断开保留登录态。** 你点断开，Bot 停掉，但 token 还在。**下次点重连，不需要重新扫码。** 这个小东西对体验提升很大——以前那种"一断就重扫"的 Bot 真的让人想砸手机。

**channel\_version 严格对齐。** 代码里有一个常量 `CHANNEL_VERSION = "2.4.6"`，严格 mirror 当前 `@tencent-weixin/openclaw-weixin` client 的版本。意思是——腾讯那边升级 client，我们这边也要跟。这种"接官方协议"的工程，跟版本是基本功。

**typing 指示器。** 微信端会显示"对方正在输入…"——这是 iLink 协议里的 `typing_status` 字段。Agent 在思考、在调工具、在写文件的时候，**你手机上能看到它真在干活，不是哑了**。

```
收到消息 → 立刻回"收到，正在处理"
        → typing 指示器亮
        → 进度气泡：🔍 查库中… ✍️ 写文件中…
        → 完成回总结
```

你永远不用面对"它是不是死了"的沉默。

### 给新读者：Obsidian CC 整体是个啥

前面讲了这么多 v0.5.0 和 v0.4.0，可能有朋友是第一次了解到 Obsidian-cc。我用三段话把它讲清楚。

**Obsidian CC 是一个 Obsidian 插件**——把 Claude Code / Codex 这两套最强 Agent 内核，连同 13 个为笔记场景调过的内置技能，整个塞进你的 Obsidian。

它干的事情，跟市面 99% 的"Obsidian AI 插件"有本质区别——

```
普通 Obsidian AI 插件：
   选中文字 → 调 GPT → 改写 / 总结 / 翻译
   ↑
   这是个工具。每次你都得明确告诉它做什么。

Obsidian CC：
   坐在你的库里 → 读你的画像 / 历史 / 记忆
   → 消化笔记 / 路由信息 / 周回顾 / 场景自适应建议
   → 主动召回 / 主动问询 / 主动沉淀
   ↑
   这是个 Agent。它知道你是谁、你在干什么、该建议什么。
```

核心结构：

```
┌──────────────────────────────────────────────────┐
│           Obsidian CC 0.5.0 能力清单              │
├──────────────────────────────────────────────────┤
│                                                   │
│  🧠 第二大脑三层记忆                              │
│     raw/ → sources/reports/ → memory/inbox/       │
│              → people/projects/wiki/decisions/     │
│                                                   │
│  🛠️ 13 个内置技能                                 │
│     消化 / 问库 / 费曼 / 帮我看看这篇 / 驾驶舱 /   │
│     路由 / 周回顾 / 记忆审核 / 记忆合并 /          │
│     画像自蒸馏 / 发现式建档 / 创建技能 / 体检      │
│                                                   │
│  ⚙️ 双引擎                                         │
│     Claude Code（Anthropic）/ Codex（OpenAI）     │
│     一键切换，共享工作区/技能/确认门               │

│     手机微信发消息 → 进库 → 回你                  │
```

这是产品价值。**不是内核价值——内核只是 Claude Code 或 Codex。产品价值是把 Agent 真正变成你的第二大脑那一整套工程。**

### 写在最后

我观察到一个趋势——

**第二大脑类产品的演化，正在从"在电脑里"走向"跟着你出门"。**

第一代是 Obsidian / Notion / Logseq——本地、强、但你得坐在电脑前。  
第二代是各种"AI 笔记助手"——云端、方便、但你的数据不在你手里。  
第三代是 Obsidian CC 这种——本地优先 + Agent + 移动入口。

```
第一代：          第二代：               第三代：
本地强            云端方便               本地 + 移动
孤岛              SaaS                   Agent + Bot
你得坐电脑前      你得信云厂商           你的库 + 你的入口
```

**第三代才是真正的"第二大脑"。** 因为它解决了一个根本问题——

**你的记忆系统，得跟着你的生活走，而不是你的生活迁就记忆系统。**

Obsidian CC v0.5.0 干的就是这件事。地铁上、咖啡馆、半夜躺床上，你的第二大脑第一次真正在场。

少即是多。随身即在场。
